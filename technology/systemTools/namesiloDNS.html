# namesilo 如何修改域名解析方式

# 1 需求

把namesilo解析域名到GitHub Pages

# 1.1 域名绑定

域名绑定也写下吧，可能以后又会用回去

# 1.1.1 更改namesilo的dns指向

如果在域名购买时，表单中的NameServers有填入dns的话，就可以直接跳到下一步啦。

如果没有的话就在右上角Manage My Domains中点击这个蓝色小按钮，删除其中的三条默认dns解析地址，再回来点击ChangeNameservers按钮。

将NameServer 1和NameServer 2改成

f1g1ns1.dnspod.net
f1g1ns2.dnspod.net

# 1.1.2 使用DNSPod解析服务

当然你想用默认或其他的dns解析服务也行。

提供dns解析的商家有很多，这里只给出DNSPod的操作步骤，但是一通百通，界面不一样，操作手法还是相同的。

添加三个数据

@ A 192.30.252.153
@ A 192.30.252.154
www CNAME username.github.io

注意：CNAME后的ip地址，千万不要使用ping命令得到的ip，因为GitHub Pages的页面ip不是固定ip，可能这阵子可以正常解析到你的页面，过阵子可能就解析到别人的页面去了！ 主机记录和记录类型说明

www：解析后的域名为 www.xxxxx.com
@：直接解析主域名 xxxxx.com
：泛解析，匹配其他所有域名 .xxxxx.com

A记录：地址记录，用来指定域名的IPv4地址（如：8.8.8.8），如果需要将域名指向一个IP地址，就需要添加A记录。
CNAME： 如果需要将域名指向另一个域名，再由另一个域名提供ip地址，就需要添加CNAME记录。

添加CNAME文件

在Hexo本地目录中的source文件夹里，添加一个命名为CNAME的无后缀文件，文件中的填写你的域名. 如果你想让地址栏的域名显示www前缀就输入www.xxxcc.com，否则输入xxxxx.com即可。 例如

www.xxxxx.com

hexo c hexo g -d

配置完后等待一会儿就可以访问啦~

参考文章：

https://amosannn.github.io/2017/06/01/github-blog-binding-domain/

# 1.2.1 更换和设置

一般而言，我们在购买域名之后，不管是Namesilo域名注册商，还是其他注册商，直接面对的问题就是域名解析。域名解析分两种方法，一种就是直接利用域名自由的解析A记录或者CNAME别名解析，一种就是更换DNS直接用第三方DNS解析，比如国内我们习惯使用DNSPOD、CLOUDXNS，国外一般用HE等等。

这里，我来分享利用更换域名DNS以及直接A记录解析的方法，我们根据自己的需要直接选择，后者A记录解析的方法生效太慢，一般海外网站用第二种方法，第二种更换DNS到国内DNS，我们中文网站使用较多。

第一、更换域名DNS

这里我们进入当前需要更换DNS的域名，看到上图，点击CHANGE，就可以更换。

这里我使用DNSPOD的免费DNS，更新完毕，我们再到DNSPOD账户中添加这个域名，以后解析域名都只要在DNSPOD等这些第三方DNS上就可以了。

第二、直接解析

我们也可以直接在NAMESILO中解析域名。

这里我们准备用自带的DNS解析域名，进入DNS中。

然后我们根据自己的需要是添加A记录呢，还是添加AAAA IPV6解析，还是CNAME别名解析。

总结，我们以上2个方法，根据我们的需要直接选择，前者用于中文站较多，后者英文站解析，毕竟域名注册商在海外，DNS比较快到海外节点。

# 1.3 使用默认解析

因为我之前购买了vultr服务器，使用的是vultr域名解析方式；这段时间用不了，所以需要更改域名解析方式为默认的，再把域名解析到GP上；

首先进入NameSilo.com, 登录自己的用户名，发现自己的域名到10月到期，还能用一段时间；

然后点击Domin Manager按钮来到设置页面，然后勾选自己的域名，点击最后一栏options选项的蓝色小球;会有一个让你更换域名DNS的选项

点击使用默认设置，把DNS解析设置成原来的；

进入域名管理后在需要修改的域名后面点击DNS解析选项（点击蓝色小球）

进入DNS解析页面后，会发现NAMESILO已经帮你生成了4个A记录，那么删掉后面两个即可（parking.namesilo.com的选项，删除这一项即可）

参考文章：

https://www.jianshu.com/p/34c2ff79af26

# 1.4 设置CNAME

到域名控制台（阿里云，腾讯云，或者其他域名供应商）找到域名列表点击解析设置，

很多人在这一步出错，或者设置完了但是访问不了（username.github.io 和 example.com 都无法访问），

仅仅添加两个CNAME类型即可，不需要管A类型，或者IP什么的(因为是解析到GP，所以不需要，直接绑定到username.github.io)

有的旧教程里面各种ping主机，填IP，统统不需要。

主机记录www和@是为了 www.example.com 和 example.com 都能访问到你的页面

这个时候你会发现还是不能访问，因为在等服务商分配DNS（可能不止10分钟），这样配置准没错，去睡个觉醒来就能访问了
